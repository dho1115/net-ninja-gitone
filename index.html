How to configure your username & password in github:

git config --global user.name [your username]

git config --global user.email [your email]

To retrieve the user name:

git config user.name

How to see which files are staged & which are committed:

git status

How to add a file to the commit:

git add [filename] or git add .

Observation about .gitignore (removing files from staging)..

So, let's say I just added something to my staging area that I did not want to. What I did was, I just went into .gitignore, inserted that file (e.g. package-lock.json) and then that file was automatically removed from staging!!!

Another way to remove files from staging.

git rm --cached filename.html (or .js or whatever extension).

What returns is rm 'filename' and you will see that file inside Changes if you click the icon on the left.

If you make changes to a file after you git add the file:

Just do another git add [changed file].

Committing a file:

git commit -m " ... ". The m stands for message.

If you see something like 75 insertions after you did git commit -m, that means there are 75 lines of new code changes.

To see all your different commits:

git log.

Type q to quit git-log.

To see the abbreviated version of your commit:

git log --oneline

How to MERGE a branch.

git merge [branch you want to merge with].

Addendum on git merge:

  1. The HEAD (Current Change) is what you have on your current branch. In other words... what you see on your VS Code now.
  2. The Incoming Change is what you are merging to. So, let's say you are on the MASTER branch and you do git merge branch-a. branch-a would be the incoming change.
  3. The best way is to make the change ON THE SCREEN. Remove the HEAD and INCOMING CHANGE comments and then delete what you want. So, if you do NOT want the incoming change, you delete the blue (incoming change).
  4. Click the "resolve in merge editor" on the right of the screen.
  5. You will see three panes. The RESULT is what the current code on that branch is.
  6. Notice the code that is highlighted in GREEN in the RESULT pane. If you click REMOVE CURRENT, what is highlighted in green in the RESULT pane will go away and the resulting code will be the code for that branch.
  7. In the upper right is the Current branch. This is how your branch originally looks like. The Result pane may or may not look like the Current pane depending on whether you click Remove Current. Once you click Remove Current, any current code that was originally in your current branch will go away.
  8. After you click Remove Current in the Result, notice that in the Current branch (upper right), there is now an Accept Current. So, if you change your mind and want to Accept Current after Removing Current in the Result, you can do it.
  9. The upper left pane is the Incoming pane. This is the code from the branch you want to merge with. For instance... let's say you are on the MASTER branch and you want to merge with Branch-A via git merge Branch-A. Branch-A would be the incoming branch. If you click ACCEPT COMBINATION in the incoming pane (upper left) and you have conflicting code (e.g.: style=red in incoming and style=green in current), the incoming will override the current (so the red will override the green).
  10. If, in the incoming pane in the upper left, you select Accept Incoming and you have conflicting code (e.g.: style=red in incoming and style=green in current for css), the red will not override the green (the current style for the branch you are on). Instead, it will add to the style, so now, you will have in your css code, both color=red AND color=green. You can then choose in your Result pane to either Remove Incoming (color) or Remove Current color (no use keeping both colors in CSS).

How to push your code up to Github

git push [that http://code you copied from Github] master. NOTE: If it's another branch, just replace master with whatever branch.

If you are NOT using VS Code and you are using the terminal to push a new branch:

  1. git remote add [origin] [your github http://url] (note: the default is "origin" but you can add any name you want).
  2. After you hit ENTER, type inside the terminal: git push [origin (or whatever name you used)] [the branch you want to push it to (e.g.: master)]. So, for instance: git push origin master.

How to CLONE a repository on Github to your computer.

  1. Inside Github, click the clone or download to reveal the HTTPS url. Copy that to use for below.
  2. Inside your TERMINAL: git clone [the HTTPS code you copied in the line above].

COMPARE & PULL REQUEST.

Assuming you are on a branch that is NOT the master branch:

  1. After you make you changes on your own branch, do a git push origin [branch you are on].
  2. Go to Github. You should now see a Compare & Pull request.
  3. Click the Compare & Pull request, add your comments. You are now requesting that the supervisor review your changes to see if he/she wants to merge it to master branch.
  4. Click Create pull request to submit this request for your changes to be merged.
  5. On the right hand side of github, you can assign reviewers.
  6. If there are conflicts (you may have to scroll to the bottom), then you may have to click "resolve conflicts" by removing the conflict markers, then making changes on the code to remove the conflicts, then complete merge.
